# 복습 퀴즈 — 13. ISR 설정

**연계 문서:** `01` Next.js 초기화, `03` Public 페이지 포팅, `13` ISR

---

1. SSG, ISR, SSR 중 가장 빠른 것은 SSR이다.
2. `revalidate = 60` 설정 시 강사가 Admin에서 수정한 내용은 최대 60초 후 공개 페이지에 반영된다.
3. Supabase 연동 전 mock 데이터를 쓰는 지금도 ISR이 정상 동작한다.
4. Link-in-Bio처럼 가끔 바뀌는 페이지에는 SSR보다 ISR이 적합하다.

---

## 정답

1. X — SSG가 가장 빠르다. 빌드 시 만든 HTML을 그대로 제공하기 때문이다.
2. O — 60초 주기로 백그라운드에서 새 HTML을 생성한다.
3. X — `fetch` 호출 없이 mock 데이터를 import하면 Next.js가 완전 정적으로 판단해 ISR로 동작하지 않는다.
4. O — 빠른 속도를 유지하면서 주기적으로 최신 데이터를 반영할 수 있다.
