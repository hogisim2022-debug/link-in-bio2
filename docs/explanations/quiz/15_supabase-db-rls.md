# 복습 퀴즈 — 14. Supabase DB & RLS

**연계 문서:** `02` 폴더 구조, `03` Public 페이지 포팅, `14` Supabase DB & RLS

---

1. `profile` 테이블은 사용자 계정당 여러 행을 가질 수 있다.
2. RLS의 SELECT 정책을 "모든 방문자 허용"으로 설정해야 공개 페이지에서 데이터를 읽을 수 있다.
3. `increment_click_count` RPC 함수를 쓰는 이유는 방문자가 RLS 때문에 직접 UPDATE할 수 없기 때문이다.
4. 마이그레이션을 적용하면 DB 변경 사항을 되돌릴 수 없다.

---

## 정답

1. X — `profile` 테이블은 계정당 딱 1개의 행만 존재한다. `id`가 로그인 계정 ID와 동일하다.
2. O — 공개 페이지(`/`)는 로그인 없이 프로필과 링크를 조회해야 하므로 SELECT는 전체 공개로 설정한다.
3. O — `SECURITY DEFINER` 속성으로 RLS를 우회해 클릭 수만 올리는 전용 창구를 만들었다.
4. X — 마이그레이션은 이력으로 남아 필요시 롤백(되돌리기)도 가능하다.
