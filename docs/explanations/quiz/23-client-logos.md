# 복습 퀴즈 — 23. 기업 로고 업로드 / 삭제 / 순서 관리

**연계 문서:** `16` Storage 버킷, `22` 프로필 이미지 업로드, `23` 기업 로고 관리

---

1. 로고를 삭제할 때 DB 행만 지우고 Storage 파일은 지우지 않아도 공개 페이지에서는 문제가 없다.
2. 드래그 중에 화면이 움직이는 것은 `@dnd-kit`이 로컬에서 처리하며, DB 업데이트는 드래그를 놓는 순간 발생한다.
3. 로고 추가 시 이미지 파일 경로를 `{userId}-{타임스탬프}.{확장자}`로 구성하면 같은 사용자가 로고를 여러 개 추가해도 파일이 덮어써지지 않는다.
4. `ClientLogoManager`에서 로고를 추가하면 공개 페이지(`/`)에도 새로고침 없이 즉시 반영된다.
5. Storage 파일을 삭제하려면 전체 URL이 아닌 버킷 내 경로(파일명)만 `remove()`에 전달해야 한다.

---

## 정답

1. X — Storage 파일이 남아 있더라도 DB에 행이 없으면 공개 페이지는 데이터를 불러오지 못한다. 하지만 Storage에 고아 파일이 계속 쌓이면 스토리지 용량을 낭비한다. 둘 다 지우는 것이 올바른 처리다.
2. O — `@dnd-kit`은 드래그 중 CSS transform으로 요소 위치를 바꾼다. `onDragEnd` 이벤트가 발생할 때 비로소 상태를 업데이트하고 DB를 호출한다.
3. O — 타임스탬프(`Date.now()`)가 항상 다른 값을 생성하므로 파일명이 겹치지 않는다. 파일 1개로 업데이트해야 하는 프로필 이미지와 달리, 로고는 여러 개를 독립적으로 유지한다.
4. X — 공개 페이지는 별도의 서버 컴포넌트(`app/page.tsx`)가 렌더링하며 ISR로 캐싱된다. 관리자 미리보기(`PreviewPane`)에는 즉시 반영되지만, 공개 페이지는 최대 60초 후 반영된다.
5. O — `supabase.storage.from("logos").remove(["user-id-1234.jpg"])` — 버킷 이름은 `from()`에 지정하고, 경로만 배열로 넘긴다.
