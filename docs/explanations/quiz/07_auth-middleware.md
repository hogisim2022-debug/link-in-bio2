# 복습 퀴즈 — 07. 인증 & Middleware

**연계 문서:** `02` 폴더 구조, `04` Admin 레이아웃, `07` 인증 Middleware

---

1. Middleware는 `/admin` 경로 접근 시 세션이 없으면 페이지를 렌더링한 뒤 `/login`으로 보낸다.
2. 쿠키 안에 저장되는 세션 ID로 서버는 로그인 상태를 판단한다.
3. 로그인 폼의 `loading` state는 제출 중 버튼을 비활성화하는 데 사용된다.
4. `.env.local`에 Supabase URL이 없으면 현재 middleware는 세션 확인 없이 `/admin`을 통과시킨다.

---

## 정답

1. X — 페이지를 렌더링하기 전에 실행된다. Middleware는 요청이 페이지에 도달하기 전에 먼저 실행된다.
2. O — 로그인 시 서버가 세션 ID를 생성하고 쿠키에 저장해 이후 요청마다 식별한다.
3. O — 제출 중 버튼을 비활성화하고 텍스트를 "로그인 중..."으로 바꾼다.
4. O — 환경변수가 없으면 middleware가 세션 확인 없이 통과시키는 개발용 bypass가 적용된다.
