# 복습 퀴즈 — 20. 링크 클릭 수 집계

**연계 문서:** `15` Supabase DB & RLS, `19` Public 페이지 연동, `20` 링크 클릭 수 집계

---

1. `increment_click_count`는 로그인한 사용자만 호출할 수 있다.
2. fire and forget 방식은 요청 결과를 확인하지 않아 집계가 실패해도 링크 이동은 정상적으로 된다.
3. `LinkItem`에 `"use client"`를 붙이면 서버에서 렌더링되지 않아 SEO에 불리하다.
4. 링크를 클릭할 때마다 `click_count`는 항상 정확히 1씩 올라가는 것이 보장된다.
5. `trackClick` 함수는 브라우저 Supabase 클라이언트(`lib/supabase/client.ts`)를 사용한다.

---

## 정답

1. X — `SECURITY DEFINER` 속성 덕분에 비로그인 방문자도 호출할 수 있다. (15번 문서 참고)
2. O — 결과를 기다리지 않으므로 집계 성공 여부와 관계없이 링크 이동이 즉시 일어난다.
3. X — `"use client"` 컴포넌트도 서버에서 초기 HTML을 렌더링한다. onClick 같은 인터랙션만 브라우저에서 처리된다.
4. X — 네트워크 오류나 동시 요청 타이밍에 따라 누락될 수 있다. 정확한 집계보다 UX 속도를 우선한 설계다.
5. O — 클릭은 브라우저(클라이언트)에서 일어나므로 브라우저용 클라이언트를 쓴다.
