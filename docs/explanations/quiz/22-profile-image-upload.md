# 복습 퀴즈 — 22. 프로필 이미지 업로드

**연계 문서:** `16` Storage 버킷, `19` Public 페이지 연동, `22` 프로필 이미지 업로드

---

1. `app/admin/page.tsx`를 서버 컴포넌트로 만들면 `useState`를 바로 사용할 수 있다.
2. Supabase Storage의 `profiles` 버킷이 public이라서 별도의 인증 없이 이미지 URL만 있으면 누구나 볼 수 있다.
3. 같은 사용자가 프로필 이미지를 두 번 바꾸면 Storage에 파일이 2개 생긴다.
4. `getPublicUrl`은 네트워크 요청 없이 URL 문자열만 조합해서 반환한다.
5. 관리자 미리보기에서 이미지가 바뀌면 공개 페이지(`/`)에도 새로고침 없이 즉시 반영된다.

---

## 정답

1. X — 서버 컴포넌트에서는 `useState` 사용이 불가능하다. 인터랙션이 필요한 부분은 별도 클라이언트 컴포넌트(`AdminClient`)로 분리해야 한다.
2. O — `profiles` 버킷은 public 설정이라 URL을 알면 누구나 이미지를 볼 수 있다. 프로필 사진은 어차피 공개 페이지에도 보여야 하므로 적절한 설정이다.
3. X — `upload(path, file, { upsert: true })`에서 path가 `{user_id}.jpg`로 고정이라 같은 경로에 덮어쓴다. 파일은 항상 1개만 유지된다.
4. O — `getPublicUrl`은 실제 네트워크 요청을 보내지 않는다. Storage URL 규칙에 따라 문자열을 조합해 반환하는 동기 함수다.
5. X — 공개 페이지는 ISR(`revalidate = 60`)로 캐싱된다. 최대 60초 후에 반영된다. 즉시 보려면 Supabase 대시보드에서 캐시를 무효화하거나 개발 서버에서 확인해야 한다.
