## DB?데이터베이스?
> 비개발자: "DB는 많이 들어봤지, 데이터+베이스? 데이터 깔아 놓은 거?"


```
엑셀 파일                     데이터베이스
─────────────────────         ────────────────────────────
파일(.xlsx)          ←→       프로젝트 (Supabase project)
시트(Sheet)          ←→       테이블 (Table)
행 (Row)             ←→       레코드 (Record)
열 (Column)          ←→       컬럼 (Column)
셀에 값 입력         ←→       INSERT (새 행 추가)
셀 내용 보기         ←→       SELECT (데이터 조회)
셀 내용 수정         ←→       UPDATE (기존 행 수정)
행 삭제              ←→       DELETE (행 제거)
```

## 핵심 개념 1: 데이터베이스란?

데이터베이스는 **데이터를 체계적으로 저장하고 꺼내 쓸 수 있는 창고**다.

엑셀과 비슷하지만 세 가지가 다르다:

| | 엑셀 | 데이터베이스 |
|--|------|------------|
| **동시 접속** | 1명만 안전하게 편집 | 수천 명이 동시에 읽고 쓸 수 있음 |
| **데이터 크기** | 수만 행이면 느려짐 | 수억 행도 빠르게 처리 |
| **보안** | 파일 암호 정도 | 행 단위로 누가 읽을 수 있는지 제어 가능 |

이 프로젝트에서는 Supabase가 데이터베이스 역할을 한다.  
3개의 테이블을 사용한다.  

[profile 테이블], [links 테이블], [client_logos 테이블]

## 핵심 개념 2: CRUD — 데이터를 다루는 4가지 동작

데이터베이스에서 할 수 있는 모든 작업은 4가지로 분류된다.

```
C — Create   새 데이터 추가    "링크를 새로 만들어"
R — Read     데이터 조회       "링크 목록 보여줘"
U — Update   기존 데이터 수정  "링크 제목 바꿔"
D — Delete   데이터 삭제       "이 링크 지워"
```

이 프로젝트(Admin, Public 페이지)에서 CRUD가 어디에 쓰이는지:

| 동작 | Admin에서 하는 일 | Public 페이지에서 하는 일 |
|------|-----------------|------------------------|
| **C** | 링크 추가 | 권한 없음 |
| **R** | 링크 목록 조회 | 방문자가 페이지 로딩 시 데이터 조회 |
| **U** | 링크 수정, 순서 변경 | 링크 클릭 시 `click_count` +1 |
| **D** | 링크 삭제 | 권한 없음 |

## 이렇게 확인하세요

Supabase 대시보드에서 직접 CRUD를 체험해본다.

1. [supabase.com](https://supabase.com) 접속 → 프로젝트 선택
2. 좌측 메뉴 **Table Editor** 클릭
3. **New table** 클릭
4. Name에 test, Columns tab에 Name: name, Type: Text 넣고 Save 클릭
5. **Insert** 버튼 클릭 → **Insert row** 클릭 → name 열에 아무 값이나 입력 후 저장 → 이것이 **Create**
6. 아무 테이블이나 선택해서 **행(Row) 목록** 확인 → 이것이 **Read**
7. 아무 행을 더블 클릭 → 값 수정 후 엔터 → 이것이 **Update**
8. 행 선택 후 삭제 버튼 클릭 → 이것이 **Delete**

## 퀴즈 정답

1. 데이터베이스는 엑셀처럼 행과 열로 데이터를 저장한다. → **O**  
   ↳ 구조는 비슷하지만 동시 접속, 처리 속도, 보안 면에서 훨씬 강력하다.

2. CRUD의 C는 데이터를 읽는(Read) 동작이다. → **X**  
   ↳ C는 Create(새 데이터 추가)다. 읽는 동작은 R(Read)이다.

3. 하나의 데이터베이스 안에 목적별로 여러 개의 표(테이블)를 만들 수 있다. → **O**  
   ↳ 이 프로젝트도 `profile`, `links`, `client_logos` 3개의 테이블을 쓴다.
