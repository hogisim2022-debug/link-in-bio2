# 로그아웃 버튼 (Supabase Auth 연동)

**완료한 task:** `2. 인증 > 로그아웃 버튼`  
**수정 파일:** `components/admin/AdminShell.tsx`

---

## 이 문서를 읽고 나면 풀 수 있어요

1. 로그아웃을 하면 브라우저의 세션 쿠키가 삭제된다.
2. 로그아웃 후 `/admin`에 다시 접근하면 middleware가 `/login`으로 보낸다.

---

## 무엇을 했나?

Admin 헤더의 로그아웃 버튼에 실제 동작을 연결했다.  
클릭 시 Supabase 세션을 종료하고 `/login` 페이지로 이동한다.

```
┌─────────────────────────────────────────────────────────┐
│  관리자                                      [로그아웃]  │  ← 클릭
└─────────────────────────────────────────────────────────┘
         ↓
   세션 쿠키 삭제
         ↓
   /login 으로 이동
         ↓
   middleware: 세션 없음 → /login 유지
```

---

## 핵심 개념: 로그아웃 흐름

로그아웃은 두 단계로 이루어진다.

1. **Supabase에 세션 종료 요청** — 서버에서 토큰을 무효화하고 브라우저 쿠키를 삭제한다.
2. **페이지 이동** — `/login`으로 보낸다. middleware가 세션 없음을 확인하고 통과시킨다.

이후 `/admin`에 다시 접근하면 세션이 없으므로 middleware가 다시 `/login`으로 보낸다.

- ❌ 과거: `"로그아웃 버튼 만들어줘"`
- ✅ 현재: `"로그아웃 버튼 클릭 시 세션 확인하고 /login으로 이동해줘"`

---

## 이렇게 확인하세요

1. 터미널에서 `npm run dev` 실행
2. 브라우저에서 `http://localhost:3000/login` 접속 후 로그인
3. `/admin` 페이지 우측 상단 **[로그아웃]** 버튼 클릭
4. `/login` 페이지로 이동되는지 확인
5. 주소창에 `http://localhost:3000/admin` 직접 입력
6. 로그인 없이 `/admin` 접근 시 다시 `/login`으로 이동되는지 확인

---

## 퀴즈 정답

1. 로그아웃을 하면 브라우저의 세션 쿠키가 삭제된다. → **O**  
   ↳ `signOut()`이 서버 토큰을 무효화하고 쿠키를 삭제한다.

2. 로그아웃 후 `/admin`에 다시 접근하면 middleware가 `/login`으로 보낸다. → **O**  
   ↳ 세션이 없으므로 middleware가 인증되지 않은 접근으로 판단하고 리다이렉트한다.
