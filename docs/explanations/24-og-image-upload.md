## 이렇게 확인하세요

1. 터미널에서 `npm run dev` 실행
2. `http://localhost:3000/admin` 접속 → 로그인
3. "SEO 설정" 섹션 하단 **[이미지 업로드]** 클릭 → 1200×630 비율 이미지 선택
4. 미리보기 이미지가 나타나는지 확인
5. Supabase 대시보드 → Storage → `og-images` 버킷에서 파일 확인
6. (배포 후에 외부 공유 가능)카카오톡 또는 슬랙에서 `http://localhost:3000/` URL 공유 시 OG 이미지 확인

## 퀴즈 정답

1. OG 이미지는 방문자가 공개 페이지에 접속할 때 화면에 표시된다. → **X**  
   ↳ OG 이미지는 링크를 SNS에 공유할 때 미리보기로 나타난다. 공개 페이지 화면에는 직접 보이지 않는다.

2. OG 이미지 권장 크기는 1200 × 630px이다. → **O**  
   ↳ 대부분의 SNS 플랫폼이 이 비율(약 1.91:1)을 기준으로 미리보기를 잘라낸다.

3. OG 이미지를 업로드하면 카카오톡이나 슬랙에 링크를 공유했을 때 미리보기에 자동으로 표시된다. → **O**  
   ↳ `generateMetadata`가 `profile.og_image_url`을 `og:image` 메타 태그에 넣기 때문에 SNS 크롤러가 자동으로 읽는다.

4. OG 이미지 업로드 로직은 프로필 이미지 업로드와 구조가 동일하고, 버킷 이름만 다르다. → **O**  
   ↳ `profiles` 버킷 → `og-images` 버킷으로만 바뀌었고, upload → getPublicUrl → DB update 흐름은 동일하다.
