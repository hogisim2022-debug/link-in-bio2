# 로그인 후 /admin 리다이렉트 (Supabase Auth 연동)

## 이 문서를 읽고 나면 풀 수 있어요

1. 보안이 중요한 middleware에서는 `getSession()` 대신 `getUser()`를 써야 한다.
2. 이미 로그인된 상태에서 `/login`에 접근하면 그냥 로그인 페이지를 보여준다.
3. 로그인 성공 후 코드에서 페이지를 새로 고침하는 이유는 서버가 새 세션 쿠키를 인식하게 하기 위해서다.

---

## 무엇을 했나?

middleware에서 세션을 검증하고, 인증 상태에 따라 `/admin` ↔ `/login`을 리다이렉트한다.

```
브라우저                   middleware.ts              Supabase Auth
   │                           │                           │
   │── GET /admin ──────────→  │                           │
   │                           │── getUser() ────────────→ │
   │                           │←─ user: null ──────────── │
   │←── 302 /login ─────────── │                           │
   │                           │                           │
   │── POST /login (폼 제출)   │                           │
   │   signInWithPassword() ──────────────────────────────→│
   │←─ 세션 쿠키 발급 ──────────────────────────────────── │
   │── router.push("/admin") → │                           │
   │── GET /admin ──────────→  │                           │
   │                           │── getUser() ────────────→ │
   │                           │←─ user: {...} ─────────── │
   │←── 200 /admin ─────────── │                           │
```

## 변경 파일

### `middleware.ts`

Supabase 클라이언트를 생성해 `getUser()`로 세션을 검증한다.

```
/admin 접근 + 로그인 안 됨  →  /login으로 이동
/login 접근 + 이미 로그인됨 →  /admin으로 이동
```

### `app/login/page.tsx`

```
이메일 + 비밀번호 → Supabase 인증 요청
  ├── 실패 → 에러 메시지 표시, 폼 유지
  └── 성공 → 세션 쿠키 발급 → /admin으로 이동
```

로그인 성공 후 코드에서 페이지를 새로 고쳐야 서버가 새 세션 쿠키를 인식하게 한다.
Next.js가 서버 컴포넌트 캐시를 조용히 갱신하는 내부 동작이다. 화면이 깜빡이거나 사용자가 뭔가 직접 할 필요가 없다.

## 핵심 개념

### `getUser()` vs `getSession()`

| | `getUser()` | `getSession()` |
|---|---|---|
| 검증 위치 | Supabase 서버 (JWT 재검증) | 로컬 쿠키만 파싱 |
| 보안 | 토큰 위·변조 감지 | 위조된 쿠키도 통과 가능 |
| 속도 | 네트워크 왕복 발생 | 즉시 |

middleware처럼 보안이 중요한 곳에서는 `getUser()`를 쓴다.

## 이렇게 확인하세요

1. 터미널에서 `npm run dev` 실행
2. 브라우저에서 주소창에 `http://localhost:3000/admin` 직접 입력 → `/login`으로 이동되는지 확인
3. 이메일, 비밀번호 입력 후 로그인 → `/admin` 페이지로 이동되는지 확인
```
    계정 생성 방법:
    1. supabase.com → 프로젝트 선택
    2. 좌측 메뉴 Authentication → Users
    3. Add user → 이메일 + 비밀번호 입력 → 저장
    4. 그 계정으로 /login에서 로그인
```   
4. 이미 로그인된 상태에서 `http://localhost:3000/login` 접속 → `/admin`으로 자동 이동되는지 확인

## 퀴즈 정답

1. 보안이 중요한 middleware에서는 `getSession()` 대신 `getUser()`를 써야 한다. → **O**  
   ↳ `getSession()`은 로컬 쿠키만 파싱해서 위조된 쿠키도 통과시킬 수 있다.

2. 이미 로그인된 상태에서 `/login`에 접근하면 그냥 로그인 페이지를 보여준다. → **X**  
   ↳ middleware가 감지해 `/admin`으로 자동 리다이렉트한다.

3. 로그인 성공 후 코드에서 페이지를 새로 고침하는 이유는 서버가 새 세션 쿠키를 인식하게 하기 위해서다. → **O**  
   ↳ 새로 고침 없이는 서버 컴포넌트가 이전 세션 상태를 유지할 수 있다.
